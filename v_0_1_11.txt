Changes in branch v_0_1_11
==================================

1) EAST training pipeline update (commit 32acec9)
- Added config options in EAST.train: score_map_shrink_ratio, lr_scheduler, lr_scheduler_params, augmentation_config.
- Expanded EASTDataset augmentations (geometric + photometric) with parameter validation.
- Added augmentation preview/list and TensorBoard preview logging.
- Added multiple LR schedulers: cosine_restart/cosine/linear/step/exponential/plateau/none.
- Updated default training settings: lr=1e-4, freeze_first=False, use_sam=False, use_lookahead=False, use_ohem=False.

Files:
- src/manuscript/detectors/_east/__init__.py
- src/manuscript/detectors/_east/dataset.py
- src/manuscript/detectors/_east/train_utils.py

2) Better TRBA import diagnostics (commit c7e4aa9)
- On training-import failure, saves original ImportError and traceback.
- Re-raises error with root-cause details (raise ... from ...).

File:
- src/manuscript/recognizers/_trba/__init__.py

3) TRBA validation size handling fix (local)
- Fixed case where val_size=0 was incorrectly treated as falsy and replaced with default 3000.
- Now fallback to 3000 happens only when val_size is None.

File:
- src/manuscript/recognizers/_trba/training/train.py

4) EAST experiment naming fix when loading local weights/checkpoints (local)
- Fixed behavior where passing a weights path from another experiment could force reusing old experiment folder name.
- Now only full-state resume checkpoints (last_state.pt / checkpoint_last.pth / checkpoint_last.pt) reuse source experiment dir.
- Weights-only files (for example best.pth) now initialize model but keep current experiment_root/model_name for a new run.

File:
- src/manuscript/detectors/_east/__init__.py

5) HTTP download retry/backoff for model weights (local)
- Added retry logic in BaseModel._download_http for transient network/server errors.
- Retries on HTTP 429/500/502/503/504 and URLError/timeout-style failures.
- Uses exponential backoff (up to 5 attempts) and cleans temporary files between attempts.

File:
- src/manuscript/api/base.py

6) Docs fix for EAST parameter name (local)
- Replaced deprecated/invalid example parameter nms_thresh with iou_threshold in Pipeline usage docs.

Files:
- docs/PIPELINE_API.md
- docs/sphinx/pipeline_usage.rst

7) Versioned static docs deployment (local)
- Updated GitHub Pages workflow to publish two documentation versions: 0.1.11 (latest) and 0.1.10 (legacy).
- Added root index redirect and versions metadata file for static hosting.
- Added docs deploy trigger for branch v_0_1_11.
- Updated Sphinx config to allow release/version override via environment variables.

Files:
- .github/workflows/docs.yml
- docs/sphinx/conf.py

8) Text-crop rotation moved from Pipeline to TRBA recognizer (local)
- Removed crop-rotation step from `Pipeline`; it now only extracts raw word crops.
- Rotation/orientation logic is now applied inside `TRBA` per crop during preprocessing.
- Rotation control is configured via `TRBA(rotate_threshold=...)` instead of `Pipeline(...)`.
- Updated pipeline docs/examples and tests to reflect the new behavior.

Files:
- src/manuscript/_pipeline.py
- src/manuscript/recognizers/_trba/__init__.py
- tests/test_pipeline_api_compatibility.py
- tests/recognizers/test_trba_infer.py
- docs/sphinx/pipeline_usage.rst

9) Docs versions switcher moved to left sidebar Contents (local)
- Added a `Versions` section in left navigation for static docs paths like `/0.1.11/en/...`.
- Versions list currently includes `0.1.11` and `0.1.10` and preserves current page path when switching.
- Replaced top-button approach for version switching with sidebar navigation.

Files:
- docs/sphinx/conf.py
- docs/sphinx/_static/version_sidebar.js
- .github/workflows/docs.yml

10) Archived static docs for 0.1.10 (local)
- Added frozen docs snapshot at `docs/sphinx/_archive/0.1.10`.
- Updated GitHub Pages workflow to stop rebuilding legacy docs.
- Workflow now copies archived `0.1.10` into `_site/0.1.10` and builds only latest docs (`0.1.11`).
- Restricted `deploy` job to `main/master`, so docs CI on feature/release branches (e.g. `v_0_1_11`) builds without failing on protected `github-pages` environment.

Files:
- docs/sphinx/_archive/0.1.10/*
- .github/workflows/docs.yml
