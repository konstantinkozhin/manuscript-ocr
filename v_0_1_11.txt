- Added config options in EAST.train: score_map_shrink_ratio, lr_scheduler, lr_scheduler_params, augmentation_config.

17) TRBA pretrain_weights=None bug fix (local)
- Fixed bug where passing `pretrain_weights=None` (train from scratch) still loaded default pretrained weights.
- Cause: `None` was not forwarded into training config due to `if pretrain_weights is not None` guard,
  so `getattr(cfg, "pretrain_weights", "default")` fell back to `"default"` and loaded release weights.
- Fix: `pretrain_weights` is now always written into config_payload regardless of its value.

File:
- src/manuscript/recognizers/_trba/__init__.py

18) EAST training stability: collage crash fix, pre-training validation, and decode alignment (local)
- Wrapped `make_collage` calls in try/except so collage failures no longer crash training.
- Added `log_collage` parameter to `EAST.train(...)` (default True) to optionally disable collage generation.
- Added `torch.cuda.empty_cache()` before collage creation to prevent CUDA OOM.
- Aligned `decode_quads_from_maps` parameters in collage visualization with inference defaults
  (score_thresh=0.6, quantization=2, iou_threshold=0.05) instead of aggressive values (0.7, 1, 0.2).
- Added pre-training validation pass (epoch 0) that computes and logs Loss/Val and Dice/Val
  before any weight updates, providing a baseline for comparing training progress.
