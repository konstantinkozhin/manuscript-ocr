Changes in v_0_1_11 relative to 0.1.10
======================================

This note lists the main user-facing changes only. Smaller fixes, test-only
changes, and internal refactors are intentionally omitted.

1) Public OCR API and pipeline redesign
- The library now uses a consistent `Page -> Page` stage contract for
  detector, layout, recognizer, and corrector components.
- Added `manuscript.api` base classes and protocols for custom stages.
- `Pipeline` was refactored around explicit stages and stage snapshots.
- Rotation handling and recognition size filtering moved out of `Pipeline`
  into `TRBA`.

2) New default pipeline and layout stage
- `Pipeline` now defaults to `YOLO(weights="yolo26x_obb_text_g1")`,
  `SimpleSorting()`, and `TRBA(weights="trba_lite_g2")`.
- Added a new `manuscript.layouts` module with the `SimpleSorting` layout.
- `layout_after` can run layout after detection, recognition, or correction.
- `manuscript.utils.organize_page` remains available as a compatibility
  wrapper.

3) YOLO detector support
- Added a new `YOLO` ONNX Runtime text detector.
- Supports preset weights, YAML config resolution, runtime provider handling,
  and local/URL/GitHub/GDrive model resolution.
- Supports both axis-aligned and rotated detections.

4) Data model and geometry updates
- The OCR data model now centers on `TextSpan`, `Line`, `Block`, and `Page`.
- Polygons with more than 4 points are now supported across the data model,
  geometry helpers, visualization, and region preparation.
- Added `page_transforms` helpers for merging and collapsing text spans.

5) Recognizer and corrector improvements
- `TRBA` and `CharLM` now operate directly on `Page` objects and accept
  optional source-image context.
- Added TRBA region preparation presets: `bbox`, `polygon_mask`, and
  `quad_warp`.
- Improved TRBA training/export support, validation, metrics, and debug
  tooling.
- Added the `trba_lite_g2` preset and expanded TRBA/EAST/CharLM
  train/export coverage.

6) EAST training and dataset improvements
- Improved COCO segmentation parsing, quad handling, and support for
  non-quad polygons.
- Expanded augmentation and preview utilities for EAST training.
- Improved resume behavior, checkpoint selection, and training stability.

7) Reliability and compatibility fixes
- HTTP model downloads now retry with backoff.
- Legacy `Word` / `words` names remain available as compatibility aliases, and
  `Page` export supports explicit schemas: `v0_1_11` and `v0_1_10`.
- Fixed Python 3.8 compatibility issues in dataclasses and type hints.
- Fixed TRBA edge cases such as `val_size=0` and
  `pretrain_weights=None`.
- Improved training reliability, including picklable collate handling and
  safer checkpoint recovery behavior.

8) Documentation and release assets
- Refreshed pipeline, API, data-model, detector, recognizer, and layout docs.
- Added versioned documentation deployment and archived `0.1.10` docs.
- Added layout API docs, model-zoo docs, and updated example notebooks.
